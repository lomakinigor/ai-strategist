## Топ-5 что чинить в первую очередь

### 1. Унифицировать стиль `/intake` с лендингом (Принцип 1: консистентность + бренд)
**Что не так:** intake живёт в Bootstrap-стиле bg-blue-600, лендинг и lead-формы — в Light Minimal (`lp-*`, navy `#1e3a8a`).
**Что делать:** переписать intake с `lp-*` классами (`lp-card` для блоков, `lp-btn-primary` для submit, navy для focus-states). 2–3 часа работы.
**Эффект:** убирает «другая программа» ощущение в самом критичном месте воронки.

### 2. Auto-generate brief для free-tier после `status='done'` (Принцип 2: минимум кликов)
**Что не так:** free-юзер открывает magic-link → /free-report → видит «пробник в обработке» → должен идти на /brief (неон-дашборд) → жмякнуть генерацию → возвращаться. 3 лишних клика + смена визуального контекста.
**Что делать:** в `notifyArtifactReady` после `status='done'` вызвать `generateBriefReport()` для tier='free', только потом отправлять email. ~30 минут кода + +$0.02 на каждый free.
**Эффект:** клиент открывает ссылку → сразу видит готовую карточку.

### 3. Поднять контраст микрокопи (Принцип 1: a11y)
**Что не так:** `#a3a3a3` на белом = 2.85:1 (WCAG минимум 4.5:1). Используется в подписях под кнопками («Анкета 5 минут. Без оплаты»), в футерах, подсказках полей.
**Что делать:** заменить `#a3a3a3` на `#6b7280` (zinc-500, ~4.83:1) везде в `lp-*` контексте. CSS-find-replace, 10 минут.
**Эффект:** доступность по WCAG AA + читабельнее для пожилой аудитории + лучше на бликующих экранах.

### 4. Добавить блок social proof (Принцип 3: реальные результаты — сейчас 0/3)
**Что не так:** нулевая доказательная база. Только «обещаем», ни одного «вот что было у клиента X».
**Что делать (даже без реальных клиентов):**
   - Использовать ГПК Репутация как анонимизированный кейс с явной пометкой «реальный клиент, имя по согласованию» — например, «Юридическая компания федерального уровня: за 1 разбор обнаружили 4 слабые точки в позиционировании и 2 свободных сегмента, которые не закрывает ни один из 5 конкурентов».
   - Блок между Method и Тарифами, 3–4 короткие цитаты или цифры (даже сначала генерируемые, потом заменить на реальные).
**Эффект:** перестаём выглядеть как «нулевой стартап без клиентов».

### 5. Прогрессивное раскрытие в `/intake` (Принцип 2: длинная форма)
**Что не так:** 15+ полей на одной странице. Анти-паттерн из вашего же CLAUDE.md.
**Что делать:** разбить на 3 шага с прогресс-баром:
   - Шаг 1 «О бизнесе»: название, отрасль, email, описание, сайт
   - Шаг 2 «Рынок»: направления, каналы, конкуренты, цели
   - Шаг 3 «Подтверждение»: AI-parse-confirmation + 152-ФЗ + submit
   Реализация — 3 секции в одной странице с tabs / wizard-state, или 3 маршрута `/intake/step1`. ~半 дня.
**Эффект:** воспринимается как «5 минут» (как обещано в копи), а не как «огромная анкета».

## Что уже хорошо (5 сильных сторон)

1. **Hero и метод** — за 3 секунды понятно что обещаем; нет «инновационной платформы».
2. **Pain-сценарии конкретные** (снижаете цены / 20 лет на сайте / конкурент уводит). Узнаваемо для ICP.
3. **FAQ с настоящими ответами**, не плейсхолдеры — большая редкость.
4. **152-ФЗ согласие сделано корректно** — не предзаполнено, ссылки в новой вкладке, отдельная плашка от parse-confirmation.
5. **Free-tier реально работает** — это не «зарегистрируйтесь чтобы посмотреть скриншоты», а полноценный пробник.

## P2 рекомендации (после основных исправлений)

- **Save form draft в localStorage** — устраняет потерю 10-минутной анкеты при F5
- **Кастомная 404 страница** в стиле `lp-*`
- **Telegram-бот для уведомлений оператора** вместо проверки Vercel logs
- **Реальный mobile-test** на iPhone/Android (Lighthouse-аудит)
- **На лендинге показать «что НЕ входит» в каждом тарифе** — повышает прозрачность (Принцип 3.3)
- **ROI-блок** на лендинге («отчёт окупается с 1 нового клиента, который придёт по новому УТП»)
- **PDF-экспорт** в `lp-*` стиле для paid tier
- **Дизайн `/brief` в Light Minimal** — устранит финальный визуальный разрыв в воронке (сейчас neon-cyberpunk)
