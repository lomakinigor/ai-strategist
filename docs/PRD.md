# PRD — ai-strategist MVP

## Суть продукта

ai-strategist — платформа для российских компаний, которая автоматически формирует стратегические материалы на основе данных о бизнесе и рынке.

**Ключевой принцип UX:** клиент заполняет минимальную форму и запускает pipeline. Система сама собирает, верифицирует данные и формирует отчёт — без длинного интервью.

## Проблема

Малый и средний бизнес РФ не может позволить себе полноценный стратегический консалтинг. Существующие инструменты либо слишком общие, либо требуют долгой ручной работы. При этом рынок, конкуренты и аудитория меняются быстро.

## Решение

Клиент заполняет минимальный intake (название, отрасль, описание, каналы). Система автоматически запускает исследовательский pipeline:

1. **Исследование бизнеса** — позиционирование, продукты, сильные и слабые стороны, конкуренты
2. **Исследование рынка и ниши** — отрасль, тренды, объём, динамика (только РФ-релевантные данные)
3. **Исследование целевой аудитории** — сегменты, потребности, поведение, боли
4. **Анализ каналов и присутствия** — где компания присутствует, насколько активна, сравнение с конкурентами
5. **Стратегия развития** — приоритеты, рекомендации с уклоном в автоматизацию и AI, суммы в рублях

> **MVP — это не чат-интервью.** Система не ведёт диалог с клиентом в процессе анализа. Уточняющие вопросы допускаются только как fallback, если критически не хватает данных.

## Первичный пользовательский flow

```
1. Intake → клиент заполняет минимальную форму (2-3 минуты)
2. Auto-research → система запускает 4 потока сбора данных параллельно (mock в MVP)
3. Validation → клиент просматривает собранные факты, может отметить неактуальные
4. Generation → система формирует структурированный отчёт через LLM + RAG
5. Report → клиент получает отчёт с маркировкой фактов
```

Шаг 3 (Validation) — опциональный в MVP. Клиент может пропустить и сразу получить отчёт.

## Целевая аудитория

- Малый и средний бизнес РФ
- Основатели, CEO, директора по развитию
- Компании, которые хотят быстро получить стратегическую картину без привлечения консультантов

## MVP — что включаем

- Минимальный intake (форма с ~5 полями)
- Auto-research pipeline с 4 потоками (mock-данные в MVP)
- Reliability Engine — верификация и маркировка всех фактов
- Validation workspace — просмотр собранных данных
- Генерация структурированного отчёта через LLM + RAG
- Workspace для просмотра готового отчёта

## MVP — что НЕ включаем

- Авторизация / многопользовательский режим (добавим позже)
- Реальные web-краулеры и API-интеграции (mock-first в MVP)
- Экспорт в PDF/DOCX (добавим после MVP)
- История нескольких отчётов (одна компания для MVP)
- Итерационный чат-интерфейс в процессе анализа

## Критерии успеха MVP

- Клиент получает полный отчёт за < 5 минут с момента отправки формы
- Каждый факт в отчёте имеет источник и RS
- Нет "выдуманных" данных без источника
- Присутствуют все 5 разделов отчёта
- Все тексты на русском языке, суммы в рублях

## Формат отчёта

Нейтральный, практичный, структурированный. Детали уточняются после запуска MVP. Базовые требования:
- 5 разделов: бизнес / рынок / аудитория / каналы / стратегия
- Маркировка ФАКТ / ГИПОТЕЗА / НЕДОСТАТОЧНО ДАННЫХ
- Указание RS для ключевых утверждений
- Пометка для мировой аналитики
- Рекомендации по автоматизации и AI с оценкой в рублях (где возможно)

## После MVP — доработка формата клиентского отчёта

**Клиентский отчёт — главный результат приложения.** MVP использует рабочий нейтральный формат. После первых тестов и появления реальных клиентских отчётов запускается отдельный этап глубокой проработки:

- структура разделов и логика блоков
- форма и стандарты клиентской подачи результата
- уровень детализации по каждому разделу
- правила для рекомендаций по автоматизации и AI
- визуальное оформление (если применимо)

Этот этап намеренно **не** входит в MVP scope — нельзя спроектировать финальный формат без реальных данных о том, что полезно клиенту.

## Ограничения

- Не является юридической или финансовой консультацией
- Точность выводов зависит от полноты загруженных данных
- Данные из закрытых источников не используются
- Мировая аналитика — только для market/niche analysis с явной пометкой
