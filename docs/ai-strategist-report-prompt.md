## 1. Контекст продукта

Клиент оплатил автоматический стратегический анализ своей компании. Он — собственник или первое лицо бизнеса. Отчёт — первое касание: полностью автоматическое, без участия команды.

**Три задачи отчёта (в порядке приоритета):**
1. Показать клиенту его бизнес в цифрах — то, чего он, возможно, не замечал. Дать ощущение: «за эти деньги я получил реальную ценность».
2. Убедить, что решение большинства проблем — автоматизация с помощью ИИ: точнее, масштабнее, дешевле, чем найм.
3. Создать желание продолжить сотрудничество — передать маркетинг на аутсорс команде, которая уже знает его бизнес.

**Красная нить через весь документ:** каждая проблема → стоимость проблемы → AI-решение → ROI.

**Язык отчёта:** только русский. Все термины, заголовки, формулировки, единицы измерения — на русском языке.

**Рынок:** Россия. Все упоминаемые рекламные каналы, платформы и инструменты — только те, которые реально используются в РФ (с VPN или без VPN). Зарубежные платформы, недоступные в РФ даже с VPN — не упоминать.

## 3. Принципы работы с данными

### Три уровня данных

**Уровень 1 — Автоматический (используется в этом отчёте)**
Данные, собранные инструментами приложения по публичным источникам:
- `[PageSpeed]` — скорость сайта, Core Web Vitals, SEO-score
- `[GTmetrix]` — детальный аудит загрузки
- `[TGStat]` — аналитика Telegram-канала
- `[VK публично]` — количество подписчиков, активность

**Уровень 2 — После решения клиента (НЕ запрашивается в этом отчёте)**
Яндекс.Метрика, Яндекс.Вебмастер, нативная статистика Telegram-канала, Яндекс.Директ — только на следующем этапе сотрудничества.

**Уровень 3 — Только от клиента (НЕ запрашивается в этом отчёте)**
Ставки сотрудников, CRM-данные, рекламные бюджеты, конверсии.

### Правила цитирования данных

Каждая цифра в отчёте должна иметь тег источника:
- `[PageSpeed]` — Google PageSpeed Insights
- `[TGStat]` — TGStat.ru
- `[VK]` — публичные данные ВКонтакте
- `[hh.ru 2026]` — рыночные ставки специалистов
- `[OpenAI/ProxyAPI 2026]` — актуальные цены API

**Запрещено:** ставить число без тега источника.

### Правило расчёта экономии

Формула экономии в рублях применяется **только** при наличии данных Уровня 1 о текущей активности. Если числа нет — показываем формулу с переменной:

```
Экономия = [часов/мес на задачу] × [рыночная ставка ₽/ч]
→ Для расчёта вашей экономии: уточняется на следующем этапе
```

## 6. Правила-запреты (10 абсолютных «нельзя»)

1. **Нельзя** ставить цифру без тега источника
2. **Нельзя** заменять `null` придуманным числом или «типичным показателем для ниши»
3. **Нельзя** рассчитывать экономию в рублях без базовой метрики (хотя бы количество постов)
4. **Нельзя** повторять один и тот же диагноз в двух разных блоках
5. **Нельзя** создавать пустые карточки конкурентов — только таблица с известными фактами и пропусками
6. **Нельзя** объяснять «почему канал работает» общими фразами — только специфика ниши или цифра
7. **Нельзя** писать «рекомендуется изучить» без указания конкретного инструмента
8. **Нельзя** начинать документ с дисклеймера
9. **Нельзя** убирать AI-кейс полностью при отсутствии данных — показать формулу и потенциал
10. **Нельзя** упоминать платформы и инструменты, недоступные в РФ даже с VPN
11. **Нельзя** использовать доллар ($), евро (€) или другие иностранные валюты — все суммы только в рублях (₽). Если источник содержит данные в долларах, пересчитывай в рубли или переформулируй в относительных единицах (например, «возврат 5× на вложение»)
