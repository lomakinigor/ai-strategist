# План разработки — ai-strategist

## Правило динамического плана

> В план можно в любой момент добавлять новые обязательные пункты.  
> Каждый новый обязательный пункт должен содержать:
> - **Описание** — что именно нужно сделать
> - **Тест-критерий** — как проверить, что сделано правильно
> - **Требование к отчёту** — что должно быть зафиксировано по итогам
>
> Добавленный пункт учитывается в текущем цикле работ.

### Этап 3 — Auto-Research Pipeline (F-004)
**Статус:** Planned

Параллельный запуск 4 потоков сбора данных после intake:

1. **Business research** — компания: продукты, конкуренты, позиционирование
2. **Market research** — рынок и ниша: объём, тренды, динамика (РФ + мировые с пометкой)
3. **Audience research** — целевая аудитория: сегменты, потребности, поведение
4. **Channels research** — каналы: активность, охват, сравнение с конкурентами

MVP: все 4 потока — mock-адаптеры. Единый интерфейс `{ data, source, date, rs, research_type }`.

**Критерий готовности:** все 4 потока возвращают mock-данные нужного формата, факты сохраняются в БД с RS

**Критерий готовности:** все факты видны, можно деактивировать, переход к генерации работает

---

### Этап 5 — RAG & AI Generation (F-006, F-007)
**Статус:** Planned

- Pipeline: активные факты → embeddings → pgvector
- Retrieval по company_id и research_type
- Генерация отчёта через Vercel AI SDK (streaming)
- System prompt включает методологию RS и запрет неверифицированных утверждений
- Fallback: запрос уточнения клиенту только при критической нехватке данных

**Критерий готовности:** отчёт генерируется по всем 5 разделам, каждый факт привязан к источнику
