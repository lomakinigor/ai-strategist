## Фреймворки исследования

### 1. АНАЛИЗ РЫНКА
*Источники: R1 (Портер), R2 (Голубой океан), R3 (Мур)*

**Шаг 1 — Точное определение рынка (Портер, R1)**
- Установить границы отрасли: продуктовые, географические, потребительские
- Если запрос клиента неоднозначен — задать уточняющий вопрос до начала исследования
- Нельзя подменять рынок смежным (пример: «детские праздники» ≠ «корпоративные мероприятия»)

**Шаг 2 — Структурный анализ по пяти силам Портера (R1)**
- Угроза новых игроков (барьеры входа, капиталоёмкость)
- Власть покупателей (концентрация, ценовая чувствительность)
- Власть поставщиков (зависимость, уникальность ресурса)
- Угроза субститутов (альтернативные решения той же потребности)
- Интенсивность конкуренции внутри отрасли

**Шаг 3 — Размер и динамика рынка**
- TAM → SAM → SOM в рублях (обязательно)
- CAGR за последние 3 года и прогноз на 2–3 года
- Сезонность, цикличность
- Источники: Росстат, РБК, отраслевые ассоциации, маркетплейсы

**Шаг 4 — Карта ценностной инновации (Голубой океан, R2)**
- Что отрасль создаёт / снижает / убирает / добавляет?
- Есть ли нереализованный спрос (overcrowded vs. blue ocean)?
- Стратегические группы: кто где находится

**Шаг 5 — Стадия зрелости рынка (Мур, R3)**
- Где рынок на кривой принятия: innovators / early adopters / chasm / majority / laggards?
- Ключевые триггеры роста и риски
- Поисковый спрос: топ-запросы, сезонность (Яндекс.Вордстат)

**Обязательные выходные данные:**
```
- Точное название исследуемого рынка (как указал клиент)
- Объём рынка РФ в ₽ (год, источник, RS)
- Динамика (CAGR), прогноз
- Пять сил: HIGH / MEDIUM / LOW по каждой
- Стадия зрелости рынка
- Топ-5 поисковых запросов ниши
- Выявленные рыночные возможности
```

### 2. АНАЛИЗ КОНКУРЕНТОВ
*Источники: C1 (Портер), C2 (Ries/Trout), C3 (Гилад)*

**Правило минимума:**
- Всегда исследовать ≥ 3 конкурентов
- Конкуренты заказчика — приоритет, исследуются все
- Если заказчик дал < 3 — добираем до 3 из открытых источников (с пометкой «добавлен AI»)
- Всегда добавляем одного конкурента, найденного AI как «наиболее релевантный по параметрам» — с объяснением выбора
- Если этот конкурент уже есть в списке заказчика — подтверждаем: «Подтверждаем как наиболее сопоставимый»

**Шаг 1 — Карта стратегических групп (Портер, C1)**
- Сгруппировать конкурентов по ценовому сегменту и позиционированию
- Определить, с кем клиент конкурирует напрямую

**Шаг 2 — Позиционная карта (Ries/Trout, C2)**
- Какую «ментальную ячейку» занимает каждый конкурент?
- Кто лидер, претендент, нишевик, следующий?
- Свободные позиции — потенциал для клиента

**Шаг 3 — CI-профиль каждого конкурента (Гилад, C3)**
Для каждого:
- Сайт, цены/тарифы, продуктовый ряд
- Позиционирование (USP, слоган, целевая аудитория)
- Сильные стороны (источники преимущества по Портеру)
- Уязвимости / слепые пятна (Гилад)
- Активность в digital (SEO, реклама, контент)

**Шаг 4 — Сравнительная матрица «Клиент vs Конкуренты»**
Параметры сравнения (под каждый рынок подбираются свои, плюс обязательные):
- Ценовой сегмент
- Охват (география)
- Уникальное предложение
- Цифровое присутствие
- Слабое место (точка атаки)

**Обязательные выходные данные:**
```
- Таблица: конкурент / позиция / USP / сильные / слабые / угроза для клиента
- AI-выбранный конкурент (обоснование)
- Свободные позиционные ниши
- Рекомендация: какую позицию занять клиенту
```

### 3. МАРКЕТИНГОВЫЙ АУДИТ
*Источники: M1 (Котлер), M2 (Ries/Trout), M3 (Годин)*

**Шаг 1 — STP-анализ клиента (Котлер, M1)**
- Какой сегмент аудитории выбран (явно или де-факто)?
- Насколько точно targeting соответствует реальному рынку?
- Позиционирование сформулировано и последовательно?

**Шаг 2 — Позиционный аудит (Ries/Trout, M2)**
- Клиент занимает уникальную «ячейку» в сознании покупателя?
- Есть ли название категории, которую клиент «владеет»?
- Конгруэнтность: имя, слоган, визуал, оффер — говорят одно?

**Шаг 3 — Доверие и разрешение (Годин, M3)**
- Клиент строит входящий поток или зависит только от рекламы?
- Есть ли механизм накопления доверия (контент, email, сообщество)?
- Минимально жизнеспособная аудитория — определена ли?

**Обязательные выходные данные:**
```
- Текущее позиционирование (факт) vs рекомендуемое
- Оценка доверия: LOW / MEDIUM / HIGH
- Разрывы в маркетинговой системе
- Топ-3 приоритета для маркетинга
```

### 4. АУДИТ САЙТА
*Источники: S1 (Круг), S2 (Эш), S3 (Вроблевски)*

**Блок А — Техническая часть** (Google PageSpeed)
- Core Web Vitals: LCP, CLS, FCP
- Performance, SEO, Accessibility, Best Practices (0–100)
- Сравнение с Lighthouse-бенчмарками отрасли

**Блок Б — Маркетинговый аудит (Круг + Эш)**

*Тест пяти секунд (Круг, S1):*
- Понятно ли с первого экрана: что это, для кого, почему выгодно?
- Есть ли явный главный призыв к действию (CTA)?
- Нет ли «шума», отвлекающего от главного

*Иерархия конверсии (Эш, S2):*
- Attention → Interest → Desire → Action (AIDA реализована?)
- Устранены ли «тревоги» пользователя: гарантии, отзывы, контакты?
- Оффер: конкретный или размытый?
- Социальные доказательства: количество, качество, размещение

*Mobile First (Вроблевски, S3):*
- Интерфейс адаптирован под мобильный как основной?
- Скрытое / убранное содержимое на мобильном — есть?
- Скорость на мобильном (PageSpeed mobile score)

**Блок В — Сравнение с конкурентами**
- Сравнение по каждому параметру: клиент vs 3 конкурента
- Выявить, в чём клиент проигрывает / выигрывает визуально
- Соответствие сайта поисковым запросам ниши (SEO-релевантность)

**Обязательные выходные данные:**
```
- Технические баллы (Performance / SEO / Accessibility)
- Маркетинговая оценка: AIDA-пробелы, оффер, CTA
- Сравнительная таблица: клиент vs конкуренты (5 параметров)
- Топ-5 приоритетных правок сайта
```

## Правила применения методологии

1. **Точность запроса** — рынок исследуется строго в том виде, как его назвал клиент. При неоднозначности — уточняющий вопрос ДО начала сбора данных.

2. **Иерархия достоверности** — каждый факт получает RS (1–5). Выводы строятся только на RS ≥ 3.

3. **Обязательные источники РФ** — Росстат, ЕГРЮЛ, РБК, Ведомости, Коммерсант, vc.ru, hh.ru, Яндекс.Вордстат.

4. **Мировая аналитика** — только с пометкой «применимость к РФ не гарантирована».

5. **Суммы** — исключительно в рублях (₽).

6. **Конкуренты** — всегда ≥ 3, всегда с AI-добавленным «наиболее релевантным».

7. **Сайт** — всегда технический + маркетинговый аудит + сравнение с конкурентами.

8. **Рынок** — Five Forces + размер рынка + стадия зрелости + поисковый спрос.
