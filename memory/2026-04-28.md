# Сессия 2026-04-28

## Тема сессии

F-007 / T-009 — AI Strategy Generation. Завершение основного MVP pipeline: intake → research → validation → strategy generation → report viewer.

## Что создали

### `src/lib/strategy/prompts.ts`
- `STRATEGY_SYSTEM_PROMPT` — системный промпт с 7 антигаллюцинационными правилами
- `buildStrategyUserPrompt(context: ResearchContext): string` — формирует user prompt из serialized RAG context с инструкцией по 5 разделам и маркировке [ФАКТ]/[ГИПОТЕЗА]/[НЕДОСТАТОЧНО ДАННЫХ]

### `src/lib/strategy/generator.ts`
- `parseSections(markdown: string): StrategySection[]` — парсит 5 разделов по паттерну `## N. Title`
- `callAnthropicAPI(systemPrompt, userPrompt): Promise<string>` — прямой fetch к `https://api.anthropic.com/v1/messages`, модель `claude-sonnet-4-6`
- `generateStrategyDraft(jobId: string): Promise<StrategyDraftResult>` — полный pipeline:
  1. Resolve companyId из research_jobs
  2. Insert report_artifact (status: generating)
  3. buildResearchContext(jobId) — RAG context из T-008
  4. callAnthropicAPI или getMockDraft
  5. Update artifact (status: done/error)
  6. parseSections → return StrategyDraftResult

### `src/lib/strategy/__tests__/generator.test.ts`
- 16 unit-тестов: parseSections (5), mock mode (5), real mode (4), callAnthropicAPI (2)
- Мокирование: buildResearchContext, prompts, getDb (select/insert/update chains), fetch

### `app/research/[id]/generate/actions.ts`
- Server Action `generateStrategyAction(formData)` → `generateStrategyDraft(jobId)` → redirect `/research/[id]/report?artifactId=...`

### `app/research/[id]/report/page.tsx`
- Server Component: загружает report_artifact, парсит разделы, отображает 5 цветных секций
- Поддерживает статусы: done (разделы), generating (pending notice), error (сообщение)
- Mock mode notice, навигация: ← Валидация / ← Исследование / Новое исследование

### `app/research/[id]/page.tsx` (изменён)
- Импорт `reportArtifacts`, `generateStrategyAction`
- При status=done: проверяет наличие существующего report_artifact
- Кнопка «Сгенерировать стратегический анализ» (form + Server Action) если артефакта нет
- Кнопка «Смотреть стратегический анализ» или «Стратегия генерируется…» если артефакт уже есть

## Архитектурные решения

- **Anthropic API — direct fetch** (не `@ai-sdk/anthropic`): пакет не в deps, pattern Perplexity уже установлен, проще не добавлять зависимость
- **Insert artifact FIRST, then buildResearchContext inside try**: любая ошибка RAG/API → artifact.status=error (запись не теряется)
- **Mock mode при отсутствии ANTHROPIC_API_KEY**: безопасный дефолт, явные [НЕДОСТАТОЧНО ДАННЫХ] метки, не скрытый fallback
- **parseSections — split по "## N."**: надёжнее RegExp с backtracking на длинных текстах
- **F-008 в docs → переименован**: T-009 фактически реализует F-007 (AI Strategy Generation), не F-008 (Strategy Workspace)
