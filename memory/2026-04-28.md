## Рефакторинг звуковых сигналов (4-я сессия)

### Проблема (было)

`beep-pre-tool.ps1` использовал эвристику: elapsed > 5 сек после последнего PostToolUse → тройной сигнал.

**Ложные срабатывания:** пауза > 5 сек по любой причине (пользователь думает, читает) → сигнал без диалога подтверждения.
**Пропуски:** диалог появился, пользователь быстро одобрил → elapsed < 5 сек → нет сигнала.

Корень проблемы: привязка к времени, а не к реальному событию `onShellCommandConfirmationRequested`.

### Решение (стало)

**`scripts/beep-pre-tool.ps1` — полная перезапись:**
- Удалена вся логика timestamp-файла и threshold
- Скрипт парсит stdin JSON и проверяет `hook_event_name`
- Тройной сигнал звучит ТОЛЬКО если `hook_event_name == "onShellCommandConfirmationRequested"` AND `tool_name == "Bash"`
- Добавлен отладочный режим: создать `%TEMP%\claude-beep-debug.flag` → лог в `%TEMP%\claude-beep-debug.log`

**`~/.claude/settings.json` — добавлен новый хук:**
- `onShellCommandConfirmationRequested` → `beep-pre-tool.ps1`
- `PreToolUse` → `beep-pre-tool.ps1` (молчит — нужен для отладки)
- `PostToolUse` → `beep-post-tool.ps1` (не изменён)
- `Stop` → `beep-done.ps1` (не изменён)

**НЕ изменены:**
- `beep-post-tool.ps1` (пишет timestamp, legacy)
- `beep-done.ps1` (двойной восходящий 660→880 Hz, Stop hook)
- `beep-wait.ps1` (не используется в hooks)

### Тест-план

#### КЕЙС А — Должен быть тройной сигнал

Условие: команда Bash требует ручного подтверждения → диалог "Allow this bash command?"

Тестовые команды (не в белом списке, требуют подтверждения):
```bash
npx drizzle-kit push          # изменение схемы БД
npm run test                  # запуск тестов
git add -A && git commit      # git-операции
npx vitest run                # vitest
```

**Ожидание:** Тройной сигнал (784→659→523) ВСЕГДА, ровно один раз, в момент появления диалога.
**Проверка:** Если сигнала нет → включить debug-флаг, посмотреть в лог какой `hook_event_name` приходит.

#### КЕЙС Б — Не должно быть тройного сигнала

| Сценарий | Ожидание |
|----------|----------|
| Bash-команда из белого списка (авто-одобрена) | Тихо |
| Edit / Write / Read / Grep / Glob | Тихо |
| Bash с `--dangerously-skip-permissions` | Тихо |
| Завершение задачи (Stop) | Только двойной восходящий 660→880 |

**Проверка Edit/Write:** выполни несколько Write операций подряд → тройного сигнала не должно быть.

#### Как проверять вручную

1. Запусти команду, которая точно требует подтверждения (например `npx drizzle-kit push`)
2. Когда появится диалог "Allow this bash command?" → должен прозвучать тройной сигнал
3. Если нет → включить debug: `New-Item "$env:TEMP\claude-beep-debug.flag"` → повторить → проверить `%TEMP%\claude-beep-debug.log`
4. В логе найти строку с event для команды из диалога
5. Если event name отличается от `onShellCommandConfirmationRequested` → скорректировать в `beep-pre-tool.ps1`
