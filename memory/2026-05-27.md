## Локальный `npm run build` падает на prerender — флак тулчейна, НЕ регрессия

**Симптом:** `next build` → `✓ Compiled successfully`, `tsc=0`, но на этапе
static-export падает:
```
TypeError: e[o] is not a function
  at app-page.runtime.prod.js ... JSON.parse ... _fromJSON
Export encountered errors on following paths: /intake/page: /intake, /page: /
=== build exit: 1 ===
```

**Диагностика (по протоколу — не повторять вслепую):** через `git stash`
собрал known-good коммит 67431f3, который Vercel задеплоил ~10 мин назад без
ошибок — он **тоже** падает локально с тем же prerender-error. Значит причина
не в коде, а в локальном окружении (Windows/Next 14 webpack-runtime RSC flight).
`rm -rf .next` и `rm -rf node_modules/.cache` не помогли.

**Вывод / правило:** при таком симптоме локальный build — НЕнадёжный сигнал.
Гейт сборки — **Vercel** (своё чистое окружение). Если Vercel-сборка падает на
prerender, деплой не промоутится → прод не ломается. Поэтому при зелёном tsc +
«Compiled successfully» и локальном prerender-флаке можно деплоить, доверяя
Vercel. Подтверждено: c536a1b собрался и задеплоился на Vercel (dpl_7zzUwGAGgf…).

Если понадобится зелёный локальный build — пробовать полную переустановку
(`rm -rf node_modules && npm ci`) или перезагрузку (вероятно file-lock
OneDrive/антивируса по пути `d:\AI\VS Code\...`).
