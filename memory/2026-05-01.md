## Архитектурные решения

- **Project-level `.claude/settings.json` отдельно от `.claude/settings.local.json`**: проектный — shared в репо, локальный — личный (gitignored). Соответствует конвенции Anthropic.
- **Шкалу `/effort` оставили свою** (`quick/normal/deep`) — переписывание ради совпадения со статьёй (`low/medium/high/xhigh/max`) — мнимая ценность.
- **Не стали добавлять `block-danger`-хук** — глобальный deny-список в `~/.claude/settings.json` уже сильный, отдельный хук дублировал бы.
- **Codex/Gemini CLI интеграция отложена** — требует доп. установок, малый эффект.

## Что не внедрено (для удалённого сервера)

- **Sandbox mode** — на Windows не работает, нужно при деплое на Linux: `~/.claude/settings.json` → `"sandbox": { "enabled": true, ... }`
- **Linux-эквиваленты PowerShell-хуков** — `.sh`-варианты `format-touched` и `log-command` для сервера
- **Hook `log-command`** — `.claude/logs/commands.tsv`, аудит-лог. Полезно перед production-деплоем.
