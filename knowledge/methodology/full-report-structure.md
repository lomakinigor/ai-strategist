# Полный отчёт для владельцев приложения — структура

**Целевая аудитория:** команда AI-Стратега (мы сами).
**Назначение:** отладка системы, расширение базы знаний, формулировка дополнительных бизнес-предложений клиенту.

## Принципиальные отличия от краткого отчёта

- **Максимум гипотез** для развития бизнеса клиента (не убираем, а наоборот — больше)
- Все RS-метки видимы ([ФАКТ], [ГИПОТЕЗА], [НЕДОСТАТОЧНО ДАННЫХ])
- AI-блок с расширенным пулом идей (5–10 вместо 3)
- Детализация по методологиям (5 сил Портера, AIDA, STP, JTBD)
- Технические данные сайта (Lighthouse, LCP, SEO)

## Секции

### 1. Бизнес — глубокий разбор
- Сильные стороны (✅) — факты + гипотезы
- Слабые стороны (❌) — факты + гипотезы
- USP-анализ
- Что улучшить (10+ пунктов)
- AI-стратег: 2–3 идеи специально для бизнеса

### 2. Сайт (если есть)
- Lighthouse score (Performance, Accessibility, Best Practices, SEO)
- AIDA-анализ
- STP-позиционирование
- Топ-5 правок с приоритетом
- AI-стратег: контент-автоматизация

### 3. Анализ рынка
- TAM / SAM / SOM (если данные есть)
- 5 сил Портера
- Стадия зрелости (по Мур)
- Ключевые тренды и драйверы
- Топ-5 поисковых запросов
- Окно возможностей (рыночные изменения)

### 4. Целевая аудитория
- Сегменты с JTBD по каждому
- Демография + психография
- Основные возражения
- AI-стратег по каждому сегменту

### 5. Каналы — топ-5 с benchmark
По каждому каналу:
- Почему работает
- Benchmark (CPL, конверсия, CPC)
- Подходит ли клиенту
- AI-стратег
- ВКЛЮЧАЯ нерелевантные с пометкой «не применимо в этой нише» — чтобы потом не возвращаться

### 6. Конкуренты — полная картотека
По каждому:
- Позиция (лидер / претендент / нишевик)
- Сильные стороны / уязвимости
- USP
- Точка атаки для клиента
- Сравнительная матрица «Клиент vs Конкуренты»
- Свободные ментальные ячейки (Ries & Trout)

### 7. AI-автоматизация — расширенный пул
- 5–10 идей с детализацией
- Включая «экспериментальные» гипотезы
- Оценка стоимости внедрения
- Канал применения
- Инструмент (API/сервис)
- Расчёт ROI с прозрачным обоснованием

### 8. Стратегия — детальный план
- Эта неделя (0 ₽) — все возможные действия
- Этот месяц — бюджет постатейно
- 3 месяца — стратегические инициативы с обоснованием

### 9. Недостаток данных
- Что не удалось собрать
- Какие источники запросить у клиента
- План доисследования

## Зачем нужен этот полный отчёт

1. **Отладка** — видим, где LLM ошибается, какие гипотезы натянутые
2. **Накопление insights для базы знаний** — что повторяется по нишам → переносится в `niches/_all.md`
3. **Аппселл-инструмент** — из расширенного пула AI-предложений выбираем что продать клиенту вторым этапом
4. **Контроль качества фактов** — RS-разметка для будущей валидации
